## "Primary keys all are real" тАФ how correctness is guaranteed

PKs are never sampled randomly: `int` PKs are sequences (1..N), `uuid` PKs are
RFC-4122 v4, business PKs use `format` templates ("CUST-######"). All are
generated by construction-unique samplers, then re-verified by the quality
engine's uniqueness check and the FK inclusion check. Person/contact values
(names, emails, phones, cities) come from dedicated realistic samplers keyed on
column names тАФ and the propose_spec system prompt now explicitly forbids the AI
from overriding them with gibberish `format` patterns (fix from live review).

## Architecture alignment

The proposed pipeline (Prompt тЖТ Research тЖТ Spec тЖТ Model тЖТ Rules тЖТ Generate тЖТ
Validate тЖТ Output) is exactly the implemented flow:

```
user prompt тЖТ research_and_generate (client web search, cited)
           тЖТ propose_spec (LLM reasoning тЖТ validated YAML)
           тЖТ apply_spec (catalog + joins + rules)
           тЖТ generate_synthetic_data (Plan IR, FK-safe, seeded)
           тЖТ run_quality_check (explained score) тЖТ preview_data
           тЖТ output: csv / parquet / sql / duckdb
```
